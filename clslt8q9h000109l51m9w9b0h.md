Cloudflare pages is the best alternative when it comes to serverless platforms like Vercel or Netlify . Because cloudflare pages is virtually free and unlimited where others are limited for 100Gb bandwidth. Also there is no limitation like no commercial use on free version like Vercel.

You can compare the pricing of [vercel](https://vercel.com/pricing), [netlify](https://www.netlify.com/pricing), [cloudflare pages](https://www.cloudflare.com/en-in/plans/developer-platform/) .

To be honest though cloudflare pages is free and unlimited but SSR (Server Side Rendering) is not supported by it which is the only visible downside of using cloudflare pages all pages are statically rendered on client machine. Cloudflare pages has unlimited bandwidth and requests for free which is great for some serious apps to casual side projects. It's better compared to paying $20 for some server side rendering and proprietary Vercel optimization.

Now let's see how to use cloudflare pages along with NEXT.JS,

### Create a [next](https://nextjs.org/docs) project with `npx create-next-app@latest`

![Terminal Next](https://cdn.hashnode.com/res/hashnode/image/upload/v1707913117934/664bb591-d118-4194-a2a3-ceace1a04c56.png align="center")

upload project to [github](https://github.com/Pratyay360/test-app)

![github page](https://cdn.hashnode.com/res/hashnode/image/upload/v1707913665642/98789bc9-d8ff-4a84-8a5a-5e3d8f92fc2b.png align="center")

Open [cloudflare homepage](https://www.cloudflare.com/) login and in dashboard go to workers and pages

![cloudflare dashoard](https://cdn.hashnode.com/res/hashnode/image/upload/v1707913912594/7f2ec47c-58cb-4eab-8f87-22388289dcfb.png align="center")

click on create application

![workers and pages](https://cdn.hashnode.com/res/hashnode/image/upload/v1707914039120/52ed78aa-17a3-4d47-9a78-2abfd5e0d252.png align="center")

click on pages and on connect to git

![create application](https://cdn.hashnode.com/res/hashnode/image/upload/v1707914223044/0cd32b05-f5fc-496f-82ba-431edd6e8194.png align="center")

Link your git repository and begin setup and choose Next.Js in the framework section

![deploy options](https://cdn.hashnode.com/res/hashnode/image/upload/v1707914357980/be577f2e-5a2f-4c29-b6c2-7b498e1cb9e5.png align="center")

leave everything as it is if you are not using environment variables. If you are using environment variables then click on the option and add your environment variables.

Hit the `Save and Deploy` button.

There will be a error while opening the generated site like

![Flag error](https://cdn.hashnode.com/res/hashnode/image/upload/v1707914929214/73c90bb4-8ed5-4d77-8646-7c48a387cc35.png align="center")

you have to copy the flag `nodejs_compat` and go to settings of the deployment and open functions window in it.

![settings](https://cdn.hashnode.com/res/hashnode/image/upload/v1707915246626/ba8527d6-4eef-48fa-a3c0-32b3f6877fa6.png align="center")

In functions window there is a option to add compatibility flags add the copied text in both Production and Preview Options.

![flags](https://cdn.hashnode.com/res/hashnode/image/upload/v1707915352071/1ef9d53d-a0c4-4cc8-97f5-a0addf38bd30.png align="center")

Also change the Compatibility date to the nearest one (if not configured automatically)

![compatibility date](https://cdn.hashnode.com/res/hashnode/image/upload/v1707915435201/e5f8a643-8c37-46ac-ac5a-4a25c5b38423.png align="center")

After Doing everything thoroughly you have to re deploy the app by clicking on retry deployment.

![](https://cdn.hashnode.com/res/hashnode/image/upload/v1707915677923/cc8eca20-0e03-42e3-849e-c04d975cf3d7.png align="center")

After doing all the steps correctly the Next.Js app will be deployed in Cloudflare Pages [link](https://test-app-kjx.pages.dev/)
